# 레지스트라(REGISTRAR) 봇 — 운영·동작 스펙

Discord 봇 **레지스트라**가 실제로 무엇을 하는지, 권한·데이터·스케줄·예외를 코드 기준으로 정리한 문서입니다. NPC·세계관은 [NPC-REFERENCES.md](./NPC-REFERENCES.md), 짧은 구현 요약은 [SPEC.md](./SPEC.md)를 참고하세요.

---

## 1. 역할 한 줄

- 길드 안에서 **등록 일정**을 공지하고, 지정 **역할** 소속 멤버의 **가용(YES) / 불가(NO)** 응답을 모읍니다.
- **응답 마감**이 되면(시간 경과 또는 관리자 강제) 공지의 버튼을 끄고 **확정 보고**를 올립니다(설정 시 PNG 첨부).
- **미제출** 명단은 해당 역할 멤버 중 아직 응답하지 않은 사람으로 계산합니다 → **Guild Members** intent·멤버 캐시/조회가 필요합니다.

---

## 2. 런타임 요구사항

| 항목           | 내용                                                                         |
| -------------- | ---------------------------------------------------------------------------- |
| Node.js        | 18+                                                                          |
| MongoDB        | 연결 URI 필수 (`MONGODB_URI`), DB명 기본 `stargate` 통합 DB (`MONGODB_DB_NAME`) |
| Discord        | Bot Token, Application(Client) ID                                            |
| Gateway Intent | **Guilds**, **GuildMembers**(특권 — 개발자 포털에서 반드시 ON)               |
| 선택           | Puppeteer/Chromium — `RESULT_CARD_IMAGE` 켤 때만 (PNG 생성)                  |

봇 기동: `connectDb()` → Discord 로그인 → `registerCommands()`(슬래시 등록) → `startCloseChecker` / `startReminderChecker`.

## 4. 슬래시 명령 트리 (`/일정`)

루트 이름은 `SCHEDULE_ROOT` = `**일정`\*\*. 서브커맨드·옵션 이름은 전부 한글 (`src/slash/ko-names.ts`).

### 4.1 공통

- 길드 전용: DM·길드 외에서는 처리하지 않음(문구로 안내).
- **서버 관리**(`ManageGuild`): `생성`, `목록`, `한눈에`, `달력`, `집계`, `마감`, `응답마감변경`, `일정변경`, `취소`.
- **일반 멤버**: `참여확인`만.

### 4.2 서브별 동작 요약

| 서브             | 입력·출력 요지                                                                                                                                                                                                                     |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **생성**         | 필수: 제목·일시·응답마감·역할. 선택: 채널(없으면 명령 실행 채널). 일시·마감은 문자열 파싱(로컬 TZ). 공지 채널에 임베드+가용/불가 버튼. DB `sessions` 삽입 후 `messageId` 동기화(실패 시 롤백 시도). 실행자에게 에페메랄 완료 안내. |
| **목록**         | 본 길드 `OPEN`만, 개수 제한·푸터에 `한눈에` 안내.                                                                                                                                                                                  |
| **한눈에**       | 월별로 접수/마감 건 요약·등록 ID·채널 링크. 길면 분할 메시지.                                                                                                                                                                      |
| **달력**         | 옵션 `월`(1–12), 선택 `채널`. 해당 월·길드의 일정을 격자 PNG로 전송(`RESULT_CARD_IMAGE` 및 권한 필요).                                                                                                                             |
| **집계**         | 옵션 `등록아이디`(선택), `이미지포함`, `채널`. 대상 건의 집계 임베드(+PNG)를 지정 채널에 게시. OPEN 다수 시 ID 필수 등 분기는 `registrar-voice` 문구대로.                                                                          |
| **참여확인**     | 본인이 YES한 건만 에페메랄. 월간 PNG는 쿨다운·설정 따름.                                                                                                                                                                           |
| **마감**         | `등록아이디` 선택. 비우면 휴리스틱으로 대상 OPEN 1건. 즉시 마감 파이프라인.                                                                                                                                                        |
| **응답마감변경** | `새응답마감` + 선택 ID. OPEN만. 공지 임베드 마감 시각 갱신 시도.                                                                                                                                                                   |
| **일정변경**     | `새일시` + 선택 ID. OPEN만. 마감이 배정보다 늦으면 자동으로 배정 1h 전 등으로 조정되는 로직 있음.                                                                                                                                  |
| **취소**         | 선택 ID. 상태 `CANCELED`, 공지를 기각 안내로 교체·버튼 비활성. 확정 보고는 없음.                                                                                                                                                   |

**등록아이디** = MongoDB `sessions` 문서의 `_id` 문자열.

### 4.3 자동완성

- `/일정 생성`의 제목·일시·응답마감·역할 필드에만 적용.
- 예시·직접입력 접두는 `Ac` 상수. 실패 시 로그 `L.autocomplete`.

## 5. 버튼(가용·불가)

- `customId` 형식: `registrar:attend:{등록ID}:yes` | `:no` (`ATTEND_BUTTON_PREFIX`).
- **OPEN**이고 **기각 아님**일 때만 토글 저장. 마감/기각 후에는 거절 메시지.
- 응답 후 공지 임베드의 가용/불가 집계 필드를 갱신(실패 시 로그).

## 6. 데이터 모델 (요약)

### 6.1 `sessions`

- `guildId`, `channelId`, `messageId`, `title`, `targetDateTime`, `closeDateTime`, `targetRoleId`, `status`, `createdBy`, `createdAt`, `updatedAt`
- `sessionStartReminder24hSent`: 배정 24h 전 리마인드 발송 여부

### 6.2 `session_responses`

- `(sessionId, userId)` 유니크, `status` YES|NO, `displayName`, 시각 필드

### 6.3 `session_logs`

- 마감·리마인드·기각 등 감사용 이벤트

### 6.4 인덱스

- `db/client.ts`의 `ensureIndexes` 참고: 마감 스캔·길드별 목록·리마인드 후보 조회·응답 역조회용.

## 7. 백그라운드 스케줄러

### 7.1 마감 (`close-checker`)

- **주기:** 약 1분.
- **대상:** `status=OPEN` 이고 `closeDateTime` ≤ 지금.
- **처리:** `executeSessionClose` — 낙관적 잠금(`OPEN`→`CLOSED`), 공지 메시지 수정(버튼 비활성·푸터), 확정 보고 채널 전송, 로그, (옵션) PNG.
- **재시도:** 레이트리밋 시 `retry_after` 기반 대기, 최대 6회.

### 7.2 리마인드 (`reminder-checker`)

- **주기:** 15분(첫 실행 45초 지연으로 마감 틱과 분리).
- **대상:** `OPEN`, 아직 리마인드 미발송, 배정 시각이 **현재로부터 24시간 이내**인 건(쿼리는 DB 모듈 참고).
- **내용:** 공지 **같은 채널**에 임베드 — 제목·배정 시각·**가용(YES) 응답자 멘션**만. YES 0명이면 발송 안 함.
- 발송 후 `sessionStartReminder24hSent=true`.

---

## 8. 미제출자 계산

- `targetRoleId` 역할의 길드 멤버 집합과 `session_responses`를 비교.
- `guild-members` 유틸: Gateway 레이트리밋 완화(캐시·간격·재시도·만료 캐시 폴백). 실패 시 명단이 어긋날 수 있어 로그 경고.

---

## 9. PNG (결과 카드·달력)

- `isResultCardImageEnabled()`가 false면 생성·첨부 생략.
- Puppeteer로 HTML 렌더 후 스크린샷. 전역 큐 + `PNG_RENDER_MIN_INTERVAL_MS`로 연속 부하 완화.
- 문구·라벨은 `registrar-voice`의 `Png` / `L` 등으로 통일.

---

## 10. 메시지·톤 (시그니처 대사)

- NPC 설정상의 **대표 대사**는 [NPC-REFERENCES.md](./NPC-REFERENCES.md)에만 두고, **봇이 자동으로 대사를 붙이지는 않음**.
- 임베드 푸터 등에는 `**REGISTRAR_SIGNATURE`\*\*(`NOVUS ORDO · REGISTRAR`) 및 업무 문구만 사용.
- 사용자에게 나가는 **한글 문구 본문**은 대부분 `src/constants/registrar-voice.ts`의 `**D`**, `**Channel**`, `**CancelEmbed**`, `**Remind**`와`src/constants/registrar.ts`의 임베드 푸터 상수에 정의됨. 아래 표는 코드 분기와 **가시성\*\*을 함께 적음.

## 11. 문구·조건·가시성 (에페메랄 여부)

### 11.1 용어

| 용어             | 의미                                                                                           |
| ---------------- | ---------------------------------------------------------------------------------------------- |
| **에페메랄**     | 슬래시/버튼 응답으로 **명령 실행자(또는 버튼 누른 사람)만** 보는 메시지.                       |
| **채널 공개**    | `channel.send` 등으로 올라간 메시지. 해당 **텍스트 채널을 볼 권한이 있는** 멤버는 누구나 열람. |
| **공지 수정**    | 기존 공지 메시지를 `edit` — 채널 열람자에게 보이는 내용이 바뀜.                                |
| **자동(스케줄)** | 봇이 타이머로 보내는 메시지 — 채널 공개.                                                       |

문구 앞에 붙는 기호: `D` 객체에서 `■` = 오류·거절, `◆` = 완료·성공, `※` = 안내.

| 상태 CANCELED                                | 에페메랄                                                                                                                                 | `editReply`: 제목·`D.resultCanceled` 조합 문자열. 그 다음 `followUp` 에페메랄: `D.resultEphemeralId`(등록번호·슬래시 안내)         |
| 상태 OPEN — 전송 성공                        | **채널 공개**: `content` + 임베드(집계용 `buildSessionEmbed`, 등록 ID 필드 숨김, 푸터 `D.resultOpenFooter`) + 선택 PNG                   | `content`에 제목 + 자동선택 시 설명 문구(코드 하드코딩 영문 설명 포함)                                                             |
| 위 성공 후                                   | 에페메랄                                                                                                                                 | `editReply`: `D.resultPosted(URL)`. `followUp`: `D.resultEphemeralId`                                                              |
| 상태 CLOSED — 전송 성공                      | **채널 공개**: `buildResultEmbed`(제목 `【확정 보고】`, 필드 가용·불가·미제출 멘션, 푸터 `EMBED_FOOTER_RESULT_CLOSED` + 서명) + 선택 PNG | 동일하게 `D.resultPosted` + `D.resultEphemeralId`                                                                                  |
| 채널 전송 실패                               | 에페메랄                                                                                                                                 | `D.resultChannelDeny` 또는 `D.resultSendFail`                                                                                      |

### 11.17 부록: 소수 상수 원문 (`registrar-voice.ts` / `registrar.ts`)

`**Channel`\*\*

- `noTarget`: 교신 채널을 확정할 수 없습니다…
- `bad`: 채널을 찾을 수 없거나…

`**CancelEmbed**`

- `title(t)`: `【기각】 {t}`
- `body`: 본 건의 가용 회신 접수는 상급 권한에 의해 **기각**되었습니다.
- `footer`: `NOVUS ORDO · REGISTRAR` (`REGISTRAR_SIGNATURE`)

`**Remind`\*\*

- `title`: 【통보】 배정 24시간 전 — 가용 회신자 명단
- `lineWhen`: 배정 시각: `<t:…:F>`
- `lineMentions`: 다음 인원은 가용으로 회신되어 있습니다: {멘션들}
- `footer`: 변경 사항은 담당 라인에 즉시 보고할 것. {서명}

`**W` (마감·기각 실패 시 에페메랄 끝에 `D.warnPrefix` 뒤로 이어 붙음)\*\*

- `noIdSkip`, `channelInaccessible`, `announceEditFail`, `resultSendFail`, `discordErr`, `logFail`, `cancelAnnounceInaccessible`

`**registrar.ts` 임베드 푸터\*\*

- `EMBED_FOOTER_OPEN`, `EMBED_FOOTER_RESULT_CLOSED`, `EMBED_FOOTER_ANNOUNCE_CLOSED` — 본문은 파일 참고.

`**D` 전체**는 함수형 항목·치환값이 많아 **11.3~11.15 표 + 소스 `registrar-voice.ts`\*\*를 정본으로 본다.

## 12. 알려진 제약·주의

- 슬래시·버튼은 **이 봇이 등록한 접두사**만 동작. 다른 봇에서 복사한 공지 버튼은 무시됨.
- 타임존: 사용자 입력 일시는 **봇 호스트 로컬 TZ** 기준 해석(문구에도 안내).
- `GUILD_ID` 운영 시: 해당 길드 외에서는 슬래시가 안 보이거나 갱신 지연될 수 있음(글로벌 등록 정책과 비교해 선택).
- PNG 사용 시 서버에 Chromium 의존 패키지 필요할 수 있음.
