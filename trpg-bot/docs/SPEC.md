# TRPG 세션 참여 체크 봇 — 구현 스펙

> 문서 작성일: 2026-03-18  
> 최종 수정: 2026-03-24 (2차 결과 카드 이미지 반영)  
> 참조: [trpg-discord-bot-plan.md](../trpg-discord-bot-plan.md)

---

## 1. 개요

TRPG 세션 일정 참여 여부를 디스코드에서 수집하고, 마감 시점에 자동으로 결과를 집계하는 봇입니다.

플랜 문서 기준 **1차 MVP**와 **2차 확장**(결과 카드 이미지 포함)을 구현한 상태이며, **3차 확장**은 미구현입니다. **4차 확장**은 플랜 문서 외 **후보 로드맵**으로만 적어 두었으며, 상용 일정 봇(Apollo·Sesh 등)에서 흔한 편의 기능과의 격차를 줄이는 방향입니다.

## 2. 플랜 대비 구현 현황

### 2-1. 구현됨 (1차 MVP)

| 플랜 항목                         | 구현 상태 | 비고                    |
| --------------------------------- | --------- | ----------------------- |
| 관리자 슬래시 명령어로 세션 생성  | ✅        | `/일정 생성` — **서버 관리** 권한 |
| 특정 채널에 세션 공지 메시지 생성 | ✅        | `채널` 옵션으로 지정   |
| 참석 / 불참 버튼 제공             | ✅        |                         |
| 사용자 응답 저장                  | ✅        | upsert, 1인 1상태       |
| 사용자가 응답을 변경할 수 있음    | ✅        | 마감 전까지             |
| 마감 시 자동 종료                 | ✅        | 1분 주기 스케줄러       |
| 최종 결과 텍스트 출력             | ✅        | 임베드 형태             |
| 대상 역할 기준 무응답자 추출      | ✅        | 최종 결과에 명단 표시   |

### 2-2. 구현됨 (2차 확장, 3차 제외)

| 플랜 항목                                      | 구현 상태 | 비고                                                                                                                                                                                           |
| ---------------------------------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 결과 카드 이미지 출력                          | ✅        | 마감 시 **최종 결과 임베드와 함께** `session-result.png` 첨부. **Puppeteer** HTML→PNG, **월간 캘린더**(세션 일시 날짜·시각 강조)+명단. `RESULT_CARD_IMAGE`가 `0` / `false` / `off` 이면 비활성 |
| 세션 시작 전 리마인드                          | ✅        | **세션 일시 24시간 이내** 구간에서 **참석(YES)만** 멘션, **15분** 폴링 (멤버 fetch 없음)                                                                                                       |
| `/일정 목록` — 진행 중 세션 목록            | ✅        | **서버 관리** 권한, 길드별 OPEN만·최근 생성 순, 에페메랄 (최대 20건 표기)                                                                                                                      |
| `/일정 한눈에` — 월별 전체 일정·ID            | ✅        | OPEN·CLOSED를 **세션 일시 순·월별**로 정리(최대 100건). PNG 없음                                                                                                                                 |
| `/일정 달력` — 월간 캘린더 PNG만            | ✅        | **서버 관리**, 정수 **`월` 1~12** (봇 로컬 **올해**). 집계 카드와 동일 격자만 출력·명단·우측 패널 없음. **채널 공개** 응답. `RESULT_CARD_IMAGE` 필요                                             |
| `/일정 집계` — 한 세션 집계·PNG(선택)       | ✅        | **서버 관리**. **채널 공개** 응답(임베드·PNG). 임베드에 **세션 ID 필드 없음** → 실행 관리자에게만 **에페메랄 followUp**으로 ID 안내. OPEN **여러 개면 ID 필수**. **`이미지포함`** 켜면 PNG          |
| `/일정 마감` — 강제 마감                   | ✅        | **서버 관리** 권한, 스케줄 마감과 동일 플로우 + 로그                                                                                                                                           |
| `/일정 응답마감변경` — 응답 마감 일시 변경    | ✅        | **서버 관리**, DB·공지 임베드 반영, 시작 리마인드 플래그 유지                                                                                                                                  |
| `/일정 일정변경` — 세션 진행 일시 변경     | ✅        | **서버 관리**, `targetDateTime`·공지 반영, **시작 24h 리마인드 플래그 초기화**                                                                                                                 |
| `/일정 취소` — 세션 취소                  | ✅        | **서버 관리** 권한, 결과 메시지 없음, 공지 임베드 교체                                                                                                                                         |
| 운영 로그 저장 (session_logs)                  | ✅        | … `EXTENDED`, `SESSION_TARGET_UPDATED`, `REMINDER_SESSION_START_24H` 등                                                                                                                        |

## 3. 현재 구현 세부 스펙

### 3-1. 전체 흐름

```
/일정 생성 → 공지 메시지 + 버튼 2개 → 사용자 버튼 클릭 → 응답 저장 → 집계 갱신
                                                                    ↓
                                              마감 시점(closeDateTime) 도달 시
                                                                    ↓
                                              버튼 비활성화 + 최종 결과 메시지 전송
```

### 3-2. 슬래시 커맨드

#### `/일정 생성` (서버 관리 권한)

- 실행: **Manage Guild** 필요. 슬래시 응답(완료·오류 안내)은 **에페메랄**(실행자만). 공지 메시지는 지정 채널에 **전체 공개**.
- `제목`·`일시`·`응답마감`·`역할` 문자열 옵션은 **자동완성**(예시 날짜·제목, 길드 역할 검색 등; 선택 없이 직접 입력도 가능)

| 옵션        | 필수 | 타입   | 설명                                    |
| ----------- | ---- | ------ | --------------------------------------- |
| `제목`      | O    | STRING | 세션명 (**autocomplete**)               |
| `일시`      | O    | STRING | 세션 진행 일시 (**autocomplete**, 직접 입력 가능) |
| `응답마감`  | O    | STRING | 응답 마감 일시 (**autocomplete**)       |
| `역할`      | O    | STRING | 참여 대상 역할 (ID·@멘션 또는 **autocomplete** 검색) |
| `채널`      | -    | CHANNEL | 공지·스레드 **목록에서 선택** (텍스트·공지·공개/비공개 스레드·공지 스레드). 미지정 시 명령을 친 채널 |

**검증 규칙**

- `응답마감` < `일시` — 응답 마감일시는 세션 일시보다 이전이어야 함
- `**응답마감` > 지금** — 마감이 이미 지난 시각이면 생성 직후 자동 마감됨. `03:50`은 **새벽** 3시 50분, 오후 3시 50분은 `**15:50`\*\* (24시간 표기 권장)
- `**일시` > 지금\*\* — 세션 일시도 미래여야 함
- 날짜 형식: `YYYY-MM-DD HH:mm` 또는 `YYYY-MM-DDTHH:mm` (봇/Node의 로컬 타임존 기준으로 해석)
- `역할` — Discord snowflake(17~19자리 숫자)만 허용. `@here`, `@everyone`은 역할이 아니므로 사용 불가

**커맨드 등록**

- 봇 ready 시 자동 등록
- `GUILD_ID` 설정 시: 해당 길드에만 등록 (즉시 반영)
- `GUILD_ID` 미설정 시: 전역 등록 (최대 1시간 캐시)

#### `/일정 목록` (서버 관리 권한)

- 이 **길드**에서 `OPEN`만, `createdAt` 내림차순 (빠른 스캔용)
- **20건** 초과 시 상위 20건만
- **마감·타월 포함 전체**는 `/일정 한눈에` 권장

#### `/일정 한눈에` (서버 관리 권한)

- `OPEN`·`CLOSED`를 **세션 일시(`targetDateTime`) 오름차순**으로 가져와 **연·월 헤더** 아래에 나열 (취소 제외)
- 각 줄: 상태, 제목, 세션 ID, 공지 채널, 세션·마감 시각(`<t:`)
- 기본 **최대 100건**; 임베드 분할·길이 한도로 잘릴 수 있음
- **PNG 없음** — 월 격자만 필요하면 `/일정 달력` + `월`(1–12). 한 세션 중심 카드는 `/일정 집계` + `이미지포함`

#### `/일정 달력` (서버 관리 권한)

| 옵션     | 필수 | 설명                                                                 |
| -------- | ---- | -------------------------------------------------------------------- |
| `월`     | O    | 정수 **1~12**. 대상 연도는 봇 프로세스 **현재 연도**(로컬 타임존) 한정 |

- `OPEN`·`CLOSED`만, `targetDateTime`이 해당 연·월에 속하는 세션을 `findSessionsByGuildInMonth`와 동일 조건으로 표시 (취소 제외)
- 응답: **채널에 공개** + `session-calendar.png` (캘린더 블록만; 우측 명단·「이번 달 일정」목록 없음)

#### `/일정 집계` (서버 관리 권한)

일반 멤버는 사용할 수 없습니다. (`Manage Guild` 필요)

| 옵션            | 필수 | 설명                                                                                                                                 |
| --------------- | ---- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `세션아이디`    | 조건부 | **OPEN 세션이 2개 이상이면 필수.** 그 외에는 비워도 됨 → OPEN이 **정확히 1개**면 그 세션, OPEN **0개**면 최근 `CLOSED` 1건 자동 선택 |
| `이미지포함`    | -    | `true`일 때만 결과 카드 PNG (`RESULT_CARD_IMAGE`·Chromium 필요). 기본은 임베드만                                               |

- **OPEN 다중**이고 `세션아이디` 없음: 채널에는 요약만 공개, **후보·ID 목록은 실행 관리자에게만 에페메랄** `followUp`
- 집계 성공(OPEN·CLOSED·CANCELED): 본문·임베드(·PNG)는 **채널 공개**, **세션 ID는 실행 관리자에게만 에페메랄** (공지/마감 채널 메시지의 임베드와는 별개)
- CANCELED: 취소 안내는 공개, ID는 위와 같이 관리자 전용

#### `/일정 마감` (서버 관리 권한)

| 옵션           | 필수 | 설명                                       |
| -------------- | ---- | ------------------------------------------ |
| `세션아이디`   | -    | 미입력 시 길드 내 가장 최근 OPEN 세션 대상 |

#### `/일정 응답마감변경` (서버 관리 권한)

응답 마감 일시(`closeDateTime`)만 바꿉니다. (구 영문 슬래시 `extend` 대체)

| 옵션             | 필수 | 설명                                       |
| ---------------- | ---- | ------------------------------------------ |
| `새응답마감`     | O    | 새 응답 마감 일시 (`일정 생성`과 동일 파싱) |
| `세션아이디`     | -    | 미입력 시 길드 내 가장 최근 OPEN 세션 대상 |

- **강제 검증 없음**: 과거 시각·세션 일시 **이후** 마감도 저장 가능 (에페메랄에 안내)
- 과거 마감 → 곧 **자동 마감**될 수 있음 / 마감 ≥ 세션 일시 → `일정변경`으로 세션 일시 조정 권장 문구
- `closeDateTime`만 변경; **세션 시작 24h 리마인드 플래그**는 유지
- 성공 시 공지 메시지 임베드 **응답 마감**·집계·버튼 갱신 (봇 **메시지 관리** 권한)
- 로그: `EXTENDED`

#### `/일정 일정변경` (서버 관리 권한)

세션 진행 일시(`targetDateTime`)를 바꿉니다.

| 옵션           | 필수 | 설명                                         |
| -------------- | ---- | -------------------------------------------- |
| `새일시`       | O    | 새 세션 일시 (`일정 생성`의 `일시`와 동일 형식) |
| `세션아이디`   | -    | 미입력 시 길드 내 가장 최근 OPEN 세션 대상   |

- **미래 필수 아님**: 과거 세션 일시도 저장 가능 (에페메랄 안내)
- `새일시`가 기존 **응답 마감 이전·동시**이면, 응답 마감을 **세션 시작 1시간 전**으로 **자동 조정** (한 번의 `update`로 `target`+`close` 반영)
- `sessionStartReminder24hSent` **초기화** (새 일정 기준으로 리마인드 재판단)
- 성공 시 공지 임베드 **세션 일시**·집계·버튼 갱신
- 로그: `SESSION_TARGET_UPDATED`

#### `/일정 취소` (서버 관리 권한)

| 옵션           | 필수 | 설명                                       |
| -------------- | ---- | ------------------------------------------ |
| `세션아이디`   | -    | 미입력 시 길드 내 가장 최근 OPEN 세션 대상 |

- 공지 메시지를 취소 안내 임베드로 바꾸고 버튼 비활성화. **최종 결과 메시지는 보내지 않음.**

### 3-3. 버튼 응답

| 라벨 | customId                      | 스타일         |
| ---- | ----------------------------- | -------------- |
| 참석 | `trpg:attend:{sessionId}:yes` | Success (녹색) |
| 불참 | `trpg:attend:{sessionId}:no`  | Danger (빨강)  |

- **1인 1상태**: 동일 유저가 다른 버튼을 누르면 기존 응답을 덮어씀 (upsert)
- **마감 전**: 버튼 클릭 시 응답 저장 후 공지 메시지 임베드의 집계 수 및 명단 실시간 갱신
- **마감 후**: 버튼 비활성화, 응답 변경 불가

#### session_logs 컬렉션 (운영 로그)

| 필드        | 타입    | 설명                 |

| CANCELED | 취소됨 (`/일정 취소`) |

### 3-5. 마감 스케줄러

- **주기**: 1분마다 (`CHECK_INTERVAL_MS = 60_000`)
- **대상**: `closeDateTime`이 지난 `OPEN` 상태 세션

**마감 시 처리**

1. 세션 상태를 `CLOSED`로 업데이트
2. 공지 메시지 수정: 버튼 비활성화, 임베드 푸터에 "마감되었습니다." 추가
3. 동일 채널에 **최종 결과 메시지** 전송 (Discord 멘션 형식)

**rate limit 대응**

- 길드 멤버 목록은 `fetchGuildMembersCached`로 TTL 캐시·재시도 (마감·`/일정 집계` 등; **리마인더는 응답 DB만 사용**). `한눈에`는 멤버 fetch 없음
- `GatewayRateLimitError` 발생 시 `retry_after`(초) 대기 후 재시도
- 마감 처리 본문은 `src/services/session-close.ts`의 `executeSessionClose`에서 공통 사용 (스케줄 마감·강제 마감).

### 3-5-1. 리마인드 스케줄러

- **주기**: **15분**마다 (`CHECK_INTERVAL_MS = 900_000`). 24시간 전후 구간에서만 의미가 있어 1분 폴링은 부하 대비 이득이 작음. 최악의 경우 알림이 한 주기(최대 15분) 늦게 나갈 수 있음.
- **첫 실행**: 마감 스케줄러(1분)와 겹침을 줄이려고 약 **45초** 지연 후 시작
- **대상**: 세션 일시가 아직 안 왔고, 시작까지 **24시간 이하**로 남은 세션 — `**OPEN` 또는 `CLOSED`\*\* (마감이 먼저 끝난 뒤에도 당일 알림 가능)
- **시점**: `targetDateTime`(세션 일시)까지 **남은 시간이 24시간 이하**이고 **아직 시작 전**일 때 1회만 (`sessionStartReminder24hSent`)
- **대상 유저**: `session_responses`에서 **참석(YES)** 인 유저만 멘션 (무응답·불참 제외). 참석자가 없으면 발송 안 함
- **채널**: 해당 세션 공지와 동일 텍스트 채널
- 멘션 대상은 DB 응답만으로 결정하므로 **길드 멤버 전체 fetch 불필요**

### 3-6. 무응답자 계산

- **대상**: `targetRoleId` 역할을 가진 멤버 (snowflake 형식이어야 함)
- **무응답**: 역할 보유자 중 `session_responses`에 기록이 없는 유저
- **표시**: 최종 결과 임베드의 "무응답" 필드에 Discord 멘션(`<@userId>`) 형식으로 나열

### 3-7. 임베드

**공지 임베드 (세션 생성 시, 버튼 클릭 시 갱신)**

| 항목      | 내용                                                                                   |
| --------- | -------------------------------------------------------------------------------------- |
| 제목      | 세션명                                                                                 |
| 필드      | 세션 일시, 응답 마감, 참석(수+멘션 명단), 불참(수+멘션 명단), **세션 ID**(관리 명령용) |
| 푸터      | "아직 응답하지 않은 인원이 있을 수 있습니다."                                          |
| 색상      | `0xc5a059` (골드 톤)                                                                   |
| 표시 형식 | Discord 멘션 `<@userId>` (결과와 동일)                                                 |

**최종 결과 임베드 (마감 시)**

| 항목      | 내용                                                                                                                                                                                                                     |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 제목      | `📋 {세션명} - 최종 결과`                                                                                                                                                                                                |
| 필드      | 세션 일시, 참석/불참/무응답 명단, **세션 ID**                                                                                                                                                                            |
| 표시 형식 | Discord 멘션 `<@userId>` — 클릭 시 유저 프로필/멘션 참조 가능                                                                                                                                                            |
| 푸터      | "마감되었습니다. 응답 변경이 불가합니다."                                                                                                                                                                                |
| 첨부 파일 | (선택) `session-result.png` — 골드 톤 카드, **같은 길드·같은 달**의 OPEN/CLOSED 세션을 월 격자에 표시(조회 중=골드, 진행=녹색 톤, 마감=회색). 조회 중 세션 **참석·불참·무응답** 명단. `RESULT_CARD_IMAGE` 비활성 시 없음 |

**임베드 필드 제한**

- Discord 임베드 필드 value 최대 1024자. 초과 시 `...`로 잘림

## 4. 미구현·참고 상세 (플랜 기준)

### 4-1. 2차 확장 — 결과 카드 이미지 (구현됨)

- **구현**: `result-card-image.ts`(렌더)·`build-session-result-card.ts`(DB에서 동일 달 세션 수집·PNG 빌드) — Puppeteer HTML→PNG.
- **캘린더**: 조회 중 세션의 `targetDateTime`이 속한 **연·월**에 대해, 같은 **길드**의 OPEN/CLOSED 세션을 `findSessionsByGuildInMonth`로 모아 격자에 표시. **조회 중** 셀은 골드, 그 외 OPEN은 녹색 톤, CLOSED는 회색 톤. 하루에 여러 세션이 있으면 시각·제목(축약) 최대 3건, 초과는 `+N`.
- **전송 시점**: `executeSessionClose`(채널 공지 스레드 `send`) 및 **`/일정 집계`** 에서 `이미지포함`이 켜진 경우 **채널 공개** `editReply` 시 임베드와 함께 첨부(CANCELED는 PNG 없음).
- **표시명**: 참석·불참은 `session_responses.displayName` 우선, 무응답은 길드 멤버 캐시의 닉네임·유저명.
- **비활성**: `RESULT_CARD_IMAGE=0` / `false` / `off` — Chromium 미설치·경량 배포 시 사용.
- **제한**: 섹션당 최대 42명까지 카드에 나열, 초과 시 “외 N명” 안내. 프로세스 종료 시 `closeResultCardBrowser()`로 브라우저 정리.

### 4-2. 3차 확장 (미구현)

- 다중 날짜 후보 세션
- 날짜별 가능 여부 응답
- 최적 날짜 자동 추천
- 캘린더형 히트맵 시각화
- 외부 웹 UI 연동

### 4-3. 4차 확장 — 후보 (상용 봇 대비, 미구현)

**배경**: 3차는 Doodle·When2meet에 가까운 **날짜 조율** 중심이다. Apollo·Sesh류에서 널리 쓰이는 **단일 일정 운영 편의**는 아래 후보에 가깝다.

| 후보 기능            | 기대 효과                                            | 구현 시 유의점                                                                 |
| -------------------- | ---------------------------------------------------- | ------------------------------------------------------------------------------ |
| **Maybe(미정)**      | “아직 모르겠다”를 공지 임베드에 반영, 집계 혼선 감소 | `session_responses.status` 확장, 최종 결과·리마인드 대상(현재 YES만) 정책 결정 |
| **반복 일정**        | 고정 요일 캠페인에 생성 반복 감소                    | 부모 템플릿 vs 인스턴스 세션, `일정변경`·취소 범위(이번만/전체)               |
| **타임존**           | 해외 멤버·서머타임 시 오해 감소                      | 길드 기본 TZ, 옵션으로 `일정 생성`에 TZ; 임베드는 Discord 타임스탬프 활용         |
| **캘린더 연동**      | 개인 일정 앱과 병행                                  | Google OAuth는 보안·토큰 보관 부담; 1단계로 iCal URL 생성만도 후보             |
| **참가 상한·대기열** | 인원 제한 테이블/원샷 세션                           | YES 순서·취소 시 승격, 마감 후 LOCK 정책                                       |
| **역할 제한 RSVP**   | 게스트 채널에서 오클 방지                            | 버튼 핸들러에서 역할 검증 vs 현재 “누르기 가능·집계만 역할”                    |

**우선순위 가이드(문서 권고만)**: Maybe → 타임존 표시 → 반복 일정 순이 **TRPG 길드**에서 체감이 큰 편. 캘린더 양방향 동기화는 범위가 커서 후순위로 두는 경우가 많다.
