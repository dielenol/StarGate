## 1. 개요

TRPG 세션 일정 참여 여부를 디스코드에서 수집하고, 마감 시점에 자동으로 결과를 집계하는 봇입니다.

플랜 문서 기준으로 **1차 MVP**를 우선 구현한 상태이며, 2차·3차 확장 기능은 미구현입니다.

---

## 2. 플랜 대비 구현 현황

### 2-1. 구현됨 (1차 MVP)


| 플랜 항목               | 구현 상태 | 비고                |
| ------------------- | ----- | ----------------- |
| 관리자 슬래시 명령어로 세션 생성  | ✅     | `/session create` |
| 특정 채널에 세션 공지 메시지 생성 | ✅     | `channel` 옵션으로 지정 |
| 참석 / 불참 버튼 제공       | ✅     |                   |
| 사용자 응답 저장           | ✅     | upsert, 1인 1상태    |
| 사용자가 응답을 변경할 수 있음   | ✅     | 마감 전까지            |
| 마감 시 자동 종료          | ✅     | 1분 주기 스케줄러        |
| 최종 결과 텍스트 출력        | ✅     | 임베드 형태            |
| 대상 역할 기준 무응답자 추출    | ✅     | 최종 결과에 명단 표시      |


### 2-2. 미구현 (2차 확장)


| 플랜 항목                                | 상태  |
| ------------------------------------ | --- |
| 마감 전 리마인드 발송                         | ❌   |
| `/session result` — 현재 집계 / 최종 결과 확인 | ❌   |
| `/session close` — 강제 마감             | ❌   |
| `/session extend` — 마감 연장            | ❌   |
| `/session cancel` — 세션 취소            | ❌   |
| 결과 카드 이미지 출력                         | ❌   |
| 운영 로그 저장 (session_logs)              | ❌   |

| 플랜 항목          | 상태  |
| -------------- | --- |
| 다중 날짜 후보 세션 생성 | ❌   |
| 날짜별 가능 여부 응답   | ❌   |
| 최적 날짜 자동 추천    | ❌   |

| 외부 웹 UI 연동     | ❌   |

## 3. 현재 구현 세부 스펙

### 3-1. 전체 흐름

```
/session create → 공지 메시지 + 버튼 2개 → 사용자 버튼 클릭 → 응답 저장 → 집계 갱신
                                                                    ↓
                                              마감 시점(closeDateTime) 도달 시
                                                                    ↓
                                              버튼 비활성화 + 최종 결과 메시지 전송
```

### 3-2. 슬래시 커맨드

#### `/session create`


| 옵션        | 필수  | 타입     | 설명                             |
| --------- | --- | ------ | ------------------------------ |
| `title`   | O   | STRING | 세션명                            |
| `date`    | O   | STRING | 세션 진행 일시 (예: 2026-03-22 20:00) |
| `close`   | O   | STRING | 응답 마감 일시 (예: 2026-03-20 23:59) |
| `role`    | O   | STRING | 참여 대상 역할 (역할 ID 또는 @역할멘션)      |
| `channel` | -   | STRING | 공지 채널 ID (미지정 시 현재 채널)         |


**검증 규칙**

- `close` < `date` — 응답 마감일시는 세션 일시보다 이전이어야 함
- 날짜 형식: `YYYY-MM-DD HH:mm` 또는 `YYYY-MM-DDTHH:mm`
- `role` — Discord snowflake(17~19자리 숫자)만 허용. `@here`, `@everyone`은 역할이 아니므로 사용 불가

**커맨드 등록**

- 봇 ready 시 자동 등록
- `GUILD_ID` 설정 시: 해당 길드에만 등록 (즉시 반영)
- `GUILD_ID` 미설정 시: 전역 등록 (최대 1시간 캐시)

### 3-3. 버튼 응답


| 라벨  | customId                      | 스타일          |
| --- | ----------------------------- | ------------ |
| 참석  | `trpg:attend:{sessionId}:yes` | Success (녹색) |
| 불참  | `trpg:attend:{sessionId}:no`  | Danger (빨강)  |


- **1인 1상태**: 동일 유저가 다른 버튼을 누르면 기존 응답을 덮어씀 (upsert)
- **마감 전**: 버튼 클릭 시 응답 저장 후 공지 메시지 임베드의 집계 수 및 명단 실시간 갱신
- **마감 후**: 버튼 비활성화, 응답 변경 불가

### 3-4. 데이터 모델

#### sessions 컬렉션


| 필드               | 타입       | 설명                       |
| ---------------- | -------- | ------------------------ |
| `_id`            | ObjectId | 세션 ID                    |
| `guildId`        | string   | 길드 ID                    |
| `channelId`      | string   | 공지 채널 ID                 |
| `messageId`      | string   | 공지 메시지 ID                |
| `title`          | string   | 세션명                      |
| `targetDateTime` | Date     | 세션 진행 일시                 |
| `closeDateTime`  | Date     | 응답 마감 일시                 |
| `targetRoleId`   | string   | 참여 대상 역할 ID              |
| `status`         | string   | OPEN | CLOSED | CANCELED |
| `createdBy`      | string   | 생성자 유저 ID                |
| `createdAt`      | Date     | 생성 시각                    |
| `updatedAt`      | Date     | 수정 시각                    |


#### session_responses 컬렉션


| 필드            | 타입      | 설명                     |
| ------------- | ------- | ---------------------- |
| `sessionId`   | string  | 세션 ID                  |
| `userId`      | string  | 디스코드 유저 ID             |
| `status`      | string  | YES | NO               |
| `displayName` | string? | 응답 시점의 표시명 (서버 닉네임 우선) |
| `respondedAt` | Date    | 응답 시각                  |
| `updatedAt`   | Date    | 수정 시각                  |


#### 세션 상태


| 상태       | 설명                       |
| -------- | ------------------------ |
| OPEN     | 응답 수집 중                  |
| CLOSED   | 마감됨                      |
| CANCELED | 취소됨 (플랜에 정의됨, 취소 로직 미구현) |


### 3-5. 마감 스케줄러

- **주기**: 1분마다 (`CHECK_INTERVAL_MS = 60_000`)
- **대상**: `closeDateTime`이 지난 `OPEN` 상태 세션

**마감 시 처리**

1. 세션 상태를 `CLOSED`로 업데이트
2. 공지 메시지 수정: 버튼 비활성화, 임베드 푸터에 "마감되었습니다." 추가
3. 동일 채널에 **최종 결과 메시지** 전송 (Discord 멘션 형식)

**rate limit 대응**

- `guild.members.fetch()` 1회만 호출 후 재사용
- `GatewayRateLimitError` 발생 시 `retry_after`(초) 대기 후 재시도

### 3-6. 무응답자 계산

- **대상**: `targetRoleId` 역할을 가진 멤버 (snowflake 형식이어야 함)
- **무응답**: 역할 보유자 중 `session_responses`에 기록이 없는 유저
- **표시**: 최종 결과 임베드의 "무응답" 필드에 Discord 멘션(`<@userId>`) 형식으로 나열

### 3-7. 임베드

**공지 임베드 (세션 생성 시, 버튼 클릭 시 갱신)**


| 항목    | 내용                                     |
| ----- | -------------------------------------- |
| 제목    | 세션명                                    |
| 필드    | 세션 일시, 응답 마감, 참석(수+멘션 명단), 불참(수+멘션 명단) |
| 푸터    | "아직 응답하지 않은 인원이 있을 수 있습니다."            |
| 색상    | `0xc5a059` (골드 톤)                      |
| 표시 형식 | Discord 멘션 `<@userId>` (결과와 동일)        |


**최종 결과 임베드 (마감 시)**


| 항목    | 내용                                            |
| ----- | --------------------------------------------- |
| 제목    | `📋 {세션명} - 최종 결과`                            |
| 필드    | 세션 일시, 참석/불참/무응답 명단                           |
| 표시 형식 | Discord 멘션 `<@userId>` — 클릭 시 유저 프로필/멘션 참조 가능 |
| 푸터    | "마감되었습니다. 응답 변경이 불가합니다."                      |


**임베드 필드 제한**

- Discord 임베드 필드 value 최대 1024자. 초과 시 `...`로 잘림

## 4. 미구현 기능 상세 (플랜 기준)

### 4-1. 2차 확장 — 관리자 명령어


| 명령어               | 플랜 설명                                      |
| ----------------- | ------------------------------------------ |
| `/session result` | 현재 집계 또는 최종 결과 확인 (session_id 또는 최근 세션 기준) |
| `/session close`  | 강제 마감                                      |
| `/session extend` | 마감 시간 연장                                   |
| `/session cancel` | 세션 취소                                      |


### 4-2. 2차 확장 — 리마인드

- **예상 시점**: 마감 24시간 전, 마감 3시간 전
- **대상**: 무응답자만 멘션

### 4-3. 2차 확장 — 기타

- 결과 카드 이미지 출력
- 운영 로그 저장 (`session_logs` 컬렉션)

### 4-4. 3차 확장

- 다중 날짜 후보 세션
- 날짜별 가능 여부 응답
- 최적 날짜 자동 추천
- 캘린더형 히트맵 시각화
- 외부 웹 UI 연동

## 5. 알려진 제한 사항


| 항목         | 설명                                  |
| ---------- | ----------------------------------- |
| 역할 외 참여 제한 | 역할 없어도 버튼 누를 수 있음 (무응답자 계산에만 역할 사용) |
| role 옵션    | `@here`, `@everyone` 사용 불가 (역할이 아님) |
| 다중 역할      | `targetRoleId` 1개만 지원               |
| 임베드 필드 길이  | 1024자 초과 시 잘림                       |
