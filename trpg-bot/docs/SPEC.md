# TRPG 세션 참여 체크 봇 — 구현 스펙

> 문서 작성일: 2026-03-18  
> 최종 수정: 2026-03-18  
> 참조: [trpg-discord-bot-plan.md](../trpg-discord-bot-plan.md)

---

## 1. 개요

TRPG 세션 일정 참여 여부를 디스코드에서 수집하고, 마감 시점에 자동으로 결과를 집계하는 봇입니다.

플랜 문서 기준 **1차 MVP**와 **2차 확장**(결과 카드 이미지 제외)을 구현한 상태이며, **3차 확장**은 미구현입니다.

## 3. 현재 구현 세부 스펙

### 3-1. 전체 흐름

```
/session create → 공지 메시지 + 버튼 2개 → 사용자 버튼 클릭 → 응답 저장 → 집계 갱신
                                                                    ↓
                                              마감 시점(closeDateTime) 도달 시
                                                                    ↓
                                              버튼 비활성화 + 최종 결과 메시지 전송
```

### 3-2. 슬래시 커맨드

#### `/session create`


| 옵션        | 필수  | 타입     | 설명                             |
| --------- | --- | ------ | ------------------------------ |
| `title`   | O   | STRING | 세션명                            |
| `date`    | O   | STRING | 세션 진행 일시 (예: 2026-03-22 20:00) |
| `close`   | O   | STRING | 응답 마감 일시 (예: 2026-03-20 23:59) |
| `role`    | O   | STRING | 참여 대상 역할 (역할 ID 또는 @역할멘션)      |
| `channel` | -   | STRING | 공지 채널 ID (미지정 시 현재 채널)         |


**검증 규칙**

- `close` < `date` — 응답 마감일시는 세션 일시보다 이전이어야 함
- **`close` > 지금** — 마감이 이미 지난 시각이면 생성 직후 자동 마감됨. `03:50`은 **새벽** 3시 50분, 오후 3시 50분은 **`15:50`** (24시간 표기 권장)
- **`date` > 지금** — 세션 일시도 미래여야 함
- 날짜 형식: `YYYY-MM-DD HH:mm` 또는 `YYYY-MM-DDTHH:mm` (봇/Node의 로컬 타임존 기준으로 해석)
- `role` — Discord snowflake(17~19자리 숫자)만 허용. `@here`, `@everyone`은 역할이 아니므로 사용 불가

**커맨드 등록**

- 봇 ready 시 자동 등록
- `GUILD_ID` 설정 시: 해당 길드에만 등록 (즉시 반영)
- `GUILD_ID` 미설정 시: 전역 등록 (최대 1시간 캐시)

#### `/session list` (서버 관리 권한)

- 이 **길드**에서 `OPEN` 상태인 세션만, `createdAt` 내림차순으로 표시
- 각 항목: 제목, 세션 ID, 공지 채널 멘션, 세션 일시·응답 마감(디스코드 `<t:` 타임스탬프)
- **20건** 초과 시 상위 20건만 임베드에 넣고 푸터에 전체 건수 표시
- 옵션 없음

#### `/session result` (서버 관리 권한)

일반 멤버는 사용할 수 없습니다. (`Manage Guild` 필요)

| 옵션           | 필수  | 설명                                             |
| ------------ | --- | ---------------------------------------------- |
| `session_id` | -   | `/session create` 완료 에페메랄에 표시되는 세션 ID. **미입력 시 이 서버(길드) 전체 기준**으로 자동 선택 (만든 사람과 **무관**) |


**`session_id` 미입력 시 자동 선택 순서**

1. 같은 길드에서 `createdAt` 기준 **가장 최근** `OPEN` 세션
2. `OPEN`이 없으면 `updatedAt` 기준 **가장 최근** `CLOSED` 세션

- OPEN: 공지와 동일 형식의 **현재 집계** 임베드(에페메랄). 응답 상단에 조회한 세션 제목·ID·자동 여부 안내
- CLOSED: **최종 결과** 임베드(에페메랄), 동일 안내
- CANCELED: 안내 문구만

#### `/session close` (서버 관리 권한)


| 옵션           | 필수  | 설명                          |
| ------------ | --- | --------------------------- |
| `session_id` | -   | 미입력 시 길드 내 가장 최근 OPEN 세션 대상 |


#### `/session edit_close` (서버 관리 권한)

응답 마감 일시(`closeDateTime`)만 바꿉니다. (구 `/session extend` 대체)

| 옵션           | 필수  | 설명                           |
| ------------ | --- | ---------------------------- |
| `new_close`  | O   | 새 응답 마감 일시 (`create`와 동일 파싱) |
| `session_id` | -   | 미입력 시 길드 내 가장 최근 OPEN 세션 대상  |


- **강제 검증 없음**: 과거 시각·세션 일시 **이후** 마감도 저장 가능 (에페메랄에 안내)
- 과거 마감 → 곧 **자동 마감**될 수 있음 / 마감 ≥ 세션 일시 → `edit_date`로 세션 일시 조정 권장 문구
- `closeDateTime`만 변경; **세션 시작 24h 리마인드 플래그**는 유지
- 성공 시 공지 메시지 임베드 **응답 마감**·집계·버튼 갱신 (봇 **메시지 관리** 권한)
- 로그: `EXTENDED`

#### `/session edit_date` (서버 관리 권한)

세션 진행 일시(`targetDateTime`)를 바꿉니다.

| 옵션           | 필수  | 설명                           |
| ------------ | --- | ---------------------------- |
| `new_date`   | O   | 새 세션 일시 (`create`의 `date`와 동일 형식) |
| `session_id` | -   | 미입력 시 길드 내 가장 최근 OPEN 세션 대상  |


- **미래 필수 아님**: 과거 세션 일시도 저장 가능 (에페메랄 안내)
- `new_date`가 기존 **응답 마감 이전·동시**이면, 응답 마감을 **세션 시작 1시간 전**으로 **자동 조정** (한 번의 `update`로 `target`+`close` 반영)
- `sessionStartReminder24hSent` **초기화** (새 일정 기준으로 리마인드 재판단)
- 성공 시 공지 임베드 **세션 일시**·집계·버튼 갱신
- 로그: `SESSION_TARGET_UPDATED`

#### `/session cancel` (서버 관리 권한)


| 옵션           | 필수  | 설명                          |
| ------------ | --- | --------------------------- |
| `session_id` | -   | 미입력 시 길드 내 가장 최근 OPEN 세션 대상 |


- 공지 메시지를 취소 안내 임베드로 바꾸고 버튼 비활성화. **최종 결과 메시지는 보내지 않음.**

### 3-3. 버튼 응답


| 라벨  | customId                      | 스타일          |
| --- | ----------------------------- | ------------ |
| 참석  | `trpg:attend:{sessionId}:yes` | Success (녹색) |
| 불참  | `trpg:attend:{sessionId}:no`  | Danger (빨강)  |


- **1인 1상태**: 동일 유저가 다른 버튼을 누르면 기존 응답을 덮어씀 (upsert)
- **마감 전**: 버튼 클릭 시 응답 저장 후 공지 메시지 임베드의 집계 수 및 명단 실시간 갱신
- **마감 후**: 버튼 비활성화, 응답 변경 불가

### 3-5. 마감 스케줄러

- **주기**: 1분마다 (`CHECK_INTERVAL_MS = 60_000`)
- **대상**: `closeDateTime`이 지난 `OPEN` 상태 세션

**마감 시 처리**

1. 세션 상태를 `CLOSED`로 업데이트
2. 공지 메시지 수정: 버튼 비활성화, 임베드 푸터에 "마감되었습니다." 추가
3. 동일 채널에 **최종 결과 메시지** 전송 (Discord 멘션 형식)

**rate limit 대응**

- 길드 멤버 목록은 `fetchGuildMembersCached`로 TTL 캐시·재시도 (마감·`/session result` 등; **리마인더는 응답 DB만 사용**)
- `GatewayRateLimitError` 발생 시 `retry_after`(초) 대기 후 재시도
- 마감 처리 본문은 `src/services/session-close.ts`의 `executeSessionClose`에서 공통 사용 (스케줄 마감·강제 마감).

### 3-5-1. 리마인드 스케줄러

- **주기**: **15분**마다 (`CHECK_INTERVAL_MS = 900_000`). 24시간 전후 구간에서만 의미가 있어 1분 폴링은 부하 대비 이득이 작음. 최악의 경우 알림이 한 주기(최대 15분) 늦게 나갈 수 있음.
- **첫 실행**: 마감 스케줄러(1분)와 겹침을 줄이려고 약 **45초** 지연 후 시작
- **대상**: 세션 일시가 아직 안 왔고, 시작까지 **24시간 이하**로 남은 세션 — **`OPEN` 또는 `CLOSED`** (마감이 먼저 끝난 뒤에도 당일 알림 가능)
- **시점**: `targetDateTime`(세션 일시)까지 **남은 시간이 24시간 이하**이고 **아직 시작 전**일 때 1회만 (`sessionStartReminder24hSent`)
- **대상 유저**: `session_responses`에서 **참석(YES)** 인 유저만 멘션 (무응답·불참 제외). 참석자가 없으면 발송 안 함
- **채널**: 해당 세션 공지와 동일 텍스트 채널
- 멘션 대상은 DB 응답만으로 결정하므로 **길드 멤버 전체 fetch 불필요**

### 3-6. 무응답자 계산

- **대상**: `targetRoleId` 역할을 가진 멤버 (snowflake 형식이어야 함)
- **무응답**: 역할 보유자 중 `session_responses`에 기록이 없는 유저
- **표시**: 최종 결과 임베드의 "무응답" 필드에 Discord 멘션(`<@userId>`) 형식으로 나열

### 3-7. 임베드

**공지 임베드 (세션 생성 시, 버튼 클릭 시 갱신)**


| 항목    | 내용                                     |
| ----- | -------------------------------------- |
| 제목    | 세션명                                    |
| 필드    | 세션 일시, 응답 마감, 참석(수+멘션 명단), 불참(수+멘션 명단), **세션 ID**(관리 명령용) |
| 푸터    | "아직 응답하지 않은 인원이 있을 수 있습니다."            |
| 색상    | `0xc5a059` (골드 톤)                      |
| 표시 형식 | Discord 멘션 `<@userId>` (결과와 동일)        |


**최종 결과 임베드 (마감 시)**


| 항목    | 내용                                            |
| ----- | --------------------------------------------- |
| 제목    | `📋 {세션명} - 최종 결과`                            |
| 필드    | 세션 일시, 참석/불참/무응답 명단, **세션 ID**                  |
| 표시 형식 | Discord 멘션 `<@userId>` — 클릭 시 유저 프로필/멘션 참조 가능 |

**임베드 필드 제한**

- Discord 임베드 필드 value 최대 1024자. 초과 시 `...`로 잘림

## 4. 미구현 기능 상세 (플랜 기준)

### 4-1. 2차 확장 — 결과 카드 이미지

- Puppeteer 등으로 결과를 이미지 카드로 출력 (미구현)

### 4-2. 3차 확장

- 다중 날짜 후보 세션
- 날짜별 가능 여부 응답
- 최적 날짜 자동 추천
- 캘린더형 히트맵 시각화
- 외부 웹 UI 연동

## 5. 알려진 제한 사항


| 항목         | 설명                                                                       |
| ---------- | ------------------------------------------------------------------------ |
| 역할 외 참여 제한 | 역할 없어도 버튼 누를 수 있음 (무응답자 계산에만 역할 사용)                                      |
| role 옵션    | `@here`, `@everyone` 사용 불가 (역할이 아님)                                      |
| 다중 역할      | `targetRoleId` 1개만 지원                                                    |
| 임베드 필드 길이  | 1024자 초과 시 잘림                                                            |
| 관리 명령      | `list` / `result` / `close` / `edit_close` / `edit_date` / `cancel` 은 **서버 관리(Manage Guild)** 권한 필요 |
