# 브리짓 케인(TEMPER) 상호작용 이미지 제작 명세

> 상태: design-proposal. 현재 프로필 이미지를 기준으로 GPT 이미지 생성에 사용할 제작 가이드다.

## 공통 고정 조건

- 기준 이미지: `public/assets/npcs/Brigid-Kane-Temper-profile.webp`
- 동일 인물 유지: 붉은 포니테일, 주근깨, 이마의 보호 고글, 검은 작업 장갑, 민소매 내열 작업복, 공구 벨트, 단련된 체격
- 화풍과 명암, 얼굴 비율, 피부 톤을 기준 이미지와 동일하게 유지
- 2:3 세로 구도, 허벅지 위까지 보이는 반신, 얼굴과 양손이 프레임 안에 들어오게 구성
- 투명 배경 PNG로 생성한 뒤 WebP로 변환. 텍스트, 로고, UI, 불꽃 배경은 이미지에 합성하지 않음
- 정지 이미지 자체에는 흔들림·기울기·확대 모션을 적용하지 않음. 상황 변화는 이미지 교체로 표현

### GPT 공통 프롬프트

```text
첨부한 브리짓 케인 캐릭터를 동일 인물로 유지해 주세요. 붉은 포니테일, 주근깨, 이마의 보호 고글, 검은 작업 장갑, 검은 민소매 내열 작업복, 공구 벨트, 근육질 체격과 흑백 중심의 러프한 애니메이션 콘셉트 아트 화풍을 그대로 보존합니다. 2:3 세로 비율의 허벅지 위 반신 컷, 얼굴과 양손이 잘리지 않게, 투명 배경, 글자와 로고 없음. 아래 상황에 맞는 표정과 손동작만 변경해 주세요.
```

## 필수 바리에이션

| UI 상태 | 파일명 | 장면 지시 |
|---|---|---|
| `inspect` 구조 판독 | `Brigid-Kane-Temper-inspect.webp` | 고글을 눈까지 내리고 한 손의 버니어 캘리퍼스로 금속 부품의 균열을 재는 모습. 웃음기 없이 눈매가 날카롭고 시선은 부품에 고정. |
| `balance` 균형 조정 | `Brigid-Kane-Temper-balance.webp` | 짧은 도검 또는 손잡이를 손바닥 위에 수평으로 얹어 무게중심을 찾는 모습. 고개를 약간 기울이고 계산하듯 집중한 표정. |
| `cart` 반출 준비 | `Brigid-Kane-Temper-cart.webp` | 작업대 위 장비의 체결부를 렌치로 조이고 다른 손으로 초크 표시를 남기는 모습. 입꼬리는 살짝 올라가지만 시선은 작업물에 고정. |
| `checkout` 검수 완료 | `Brigid-Kane-Temper-checkout.webp` | 검수표 또는 금속 태그에 TEMPER 각인을 마친 뒤 작은 검사 망치를 어깨에 걸친 모습. 정면을 보며 자신감 있게 미소. |
| `blocked` 반출 거부 | `Brigid-Kane-Temper-blocked.webp` | 웃음기가 완전히 사라지고 고글을 눈까지 내린 채 한 손바닥으로 정지 신호. 다른 손에는 반출 불가 태그. 분노보다 냉정한 존댓말이 느껴지는 표정. |
| `idle` 단조 중 | `Brigid-Kane-Temper-idle.webp` | 모루 위의 작은 금속 부품을 망치로 내려치기 직전의 자세. 얼굴은 옆으로 향하고 작업에 몰입. UI에서 별도 불꽃 효과가 들어가므로 이미지 자체에는 불꽃을 그리지 않음. |

현재 `Brigid-Kane-Temper-profile.webp`는 고글을 들어 올리며 웃는 포즈라 `welcome`과 기본 프로필에 그대로 사용한다.

## 선택 바리에이션

| UI 상태 | 파일명 | 장면 지시 |
|---|---|---|
| 토와스키 열람 연계 | `Brigid-Kane-Temper-referral.webp` | 토와스키 열람표를 한 손에 들고 못마땅한 듯 옆눈으로 읽으면서도 규격 수치는 인정하는 작은 비웃음. 다른 손에는 측정 초크. 총기나 탄약은 들지 않음. |

`referral`은 할인 상태를 강하게 구분하고 싶을 때만 추가한다. 우선순위는 필수 6종보다 낮다.

## 생성 후 체크

- 얼굴, 포니테일 방향, 고글 형태, 장갑 길이, 작업복 절개가 컷마다 달라지지 않았는지 확인
- 손가락 수, 공구 결합, 칼날 방향, 캘리퍼스 구조가 무너지지 않았는지 확인
- `blocked`에 미소가 남지 않았는지, `checkout`이 과도한 승리 포즈가 아닌지 확인
- 투명 여백과 인물 크기가 모든 파일에서 같아 이미지 전환 시 위치가 튀지 않는지 확인
- 최종 파일을 전달받은 뒤 `TemperMood`별 이미지 매핑을 연결하고 데스크톱·모바일 HUD 크롭을 검증
